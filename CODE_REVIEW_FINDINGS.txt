ISSUE: Incomplete Event Propagation Fix on Modal Close Buttons
--------------------------------------------------------------
Severity: MEDIUM
Files: frontend/js/app.js (4 locations)

PROBLEM:
The CHANGELOG states "Event propagation on modal close and back buttons — clicking 
the close (X) or back buttons in modals previously bubbled the event to parent 
listeners, which could trigger accidental navigation or re-open the modal. 
stopPropagation() is now called before invoking the handler."

However, this fix was ONLY applied to the game detail modal's close button 
(ui.js:771). Four other modals are MISSING stopPropagation():

  1. Players Modal (line ~1773)
  2. Shortcuts Modal (line ~2263)
  3. Game Night Modal (line ~2804)
  4. Share Modal (line ~3711)

IMPACT:
Clicking close (X) buttons in these modals may trigger parent event listeners,
causing accidental navigation or modal re-opening. This is a regression.

REMEDY:
Add e.stopPropagation() to all modal close button handlers.

Current code (INCORRECT):
    inner.querySelector('#players-modal-close').addEventListener('click', close);

Fixed code (CORRECT):
    inner.querySelector('#players-modal-close').addEventListener('click', (e) => {
      e.stopPropagation();
      close();
    });

Apply this pattern to ALL 4 modal close button handlers.

Files to modify:
  - frontend/js/app.js (lines ~1773, ~2263, ~2804, ~3711)

Estimated fix time: 5-10 minutes
